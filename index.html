"What has changed?"

What has happened to Java?

Situation is unclear / changing...

But this is what I found so far...

No production use!

"Now that Java 11 is released, we can see that Oracle JDK can be downloaded for $free. However, the license has changed, explicitly ruling out use in a production environment (without paying)."

New six-monthly releases

Java 8March 2014January 2019 (for commercial use)
Java 9Sept 2017March 2018

Java 11Sept 2018March 2019
Java 12March 2019Sept 2019

Do I have to care?

Apparently YES

The trap is as follows:

  • Download Oracle JDK (because that is what you've always done)
  • Use it in production (because you didn't realise the license changed)
  • Get a nasty phone call from Oracle's license enforcement teams demanding lots of money

The solution

Options

If you want to stay on Java 8 after January 2019, here are the choices as I see them:

  1. Don't care about security
    -> up to you ;-)
  2. Rely on Operating System updates
    -> I think this does not apply to Windows (for Linux this is a variation of point 6)
  3. Pay for support
    -> this is the right option if the customer has decided to use and pay for a supported/commercial product
  4. Use the non-commercial build
    -> illegal
  5. Build OpenJDK yourself
    -> not realistic for most customers (questional added value compared to point 6)
  6. Use the builds from AdoptOpenJDK
    -> IMHO the way to go for most cases

Conclusion

The solution is relatively simple:

use AdoptOpenJDK for most cases

But discuss with customer: "the JDK you use should now be a choice you actively make"

Differences between Oracle JDK and AdoptOpenJDK

Now OpenJDK is the base of most Java JDKs (including Oracle's)

For most cases: from Java 11, developers can treat Oracle JDK and OpenJDK as being equivalent
